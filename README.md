# Création d’un système d’information hospitalier

La conception d’un réseau hospitalier nécessite une architecture capable de soutenir en continu les activités critiques de l’établissement.

## 1. Architecture du système d’information

La première partie du rapport présente l’architecture du système d’information, en détaillant :

- la répartition des équipements ;
- l’organisation logique du réseau ;
- le plan d’adressage ;
- le cœur de réseau ;
- les interconnexions ;
- les DMZ ;
- le Wi-Fi ;
- le lien Internet.

Cette section établit les fondations techniques du projet et décrit la manière dont les différents bâtiments et services sont intégrés dans l’infrastructure.

## 2. Sécurisation du système d’information

La seconde partie est consacrée à la sécurisation du système d’information, un aspect essentiel pour protéger les données sensibles et garantir la continuité des services du SI. Elle présente notamment :

- les principes de segmentation ;
- l’usage de bastions d’administration et de postes dédiés ;
- la gestion des accès via l’annuaire ;
- les ACL ;
- les règles de firewall nécessaires pour maîtriser les flux et limiter les risques.

## 3. Pistes d’évolution

Enfin, la troisième partie propose des pistes d’évolution, notamment :

- la redondance opérateur ;
- l’ajout d’un lien 5G de secours ;

afin d’améliorer la résilience globale du réseau.

## Annexes

Le rapport se conclut par un ensemble d’annexes regroupant les éléments techniques indispensables à la compréhension et à la mise en œuvre du projet, tels que :

- le schéma logique ;
- la matrice de flux ;
- le plan d’adressage ;
- les configurations des équipements.
